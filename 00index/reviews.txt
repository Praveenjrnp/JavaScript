🔹Que 1️⃣ : Why 'let' was introduced if 'var' already existed?
        : Why 'var' leaks outside block but 'let' does not?

In early JavaScript, only "var" was used to declare variables. But it had some problems:

Problems with var:
* Function scoped, not block scoped
* Can be redeclared accidentally
* Can create bugs in loops and conditions


Example with var:--------------------------------------------------

if(true){
  var x = 10;
}
console.log(x); // 10 (accessible outside block)
------------------------------------------------------------------ 

=> So "let" was introduced in ES6 to fix this.

Example with let:----------------------------------

if(true){
  let x = 10;
}
console.log(x); // Error (block scope)
------------------------------------------------------

Advantages of let:

* Block scoped
* Cannot redeclare in same scope
* Safer for loops and conditions

🔹Que 2️⃣ Why const allows changing object properties?

const means the reference cannot change, not the content.

Equal-to (=) use krke nayi value nahi daal skte, but object k andar ek property ki value baadalte hain to that is allowed.

Example:--------------------------------
const obj = {name: "Nick"};
obj.name = "Rahul"; // allowed
name = {} // not allowed
console.log(obj);
-----------------------------------------
Works because we did not change the object reference, only its property.


But this is not allowed:-----------------
const obj = {name: "Nick"};
obj = {name: "Rahul"}; // Error
-----------------------------------------
Because now we are reassigning the object.

🔹Que 3️⃣ : Object.freeze() in JavaScript.

Object.freeze() kisi object ko completely immutable (change-proof) bana deta hai.

Iske baad:

❌ properties change nahi ho sakti

❌ new properties add nahi ho sakti

❌ properties delete nahi ho sakti


example -----------------------------------
const obj = {
  name: "Nick",
  age: 21
};

Object.freeze(obj);

obj.name = "Rahul";   // ❌ change nahi hoga
obj.city = "Delhi";   // ❌ add nahi hoga
delete obj.age;       // ❌ delete nahi hoga

console.log(obj);
-------------------------------------------
Output: { name: "Nick", age: 21 }
Object same hi rahega.

🔹Que 10 : What is closure? When is it created?
  🔹 What is a Closure?
👉 Jab ek func dusre func ko return kar rha ho AND andar wala func, bahar wale (Parent) func  ke kisi variable ko use kar rha ho.

ex 1: ----------------------------------------
function abcd(){
  let val = 0;
  return function(){
    console.log(val);
  }
}
--------------------------------------------

ex 2: ------------------------------------------------------
function outer() {
  let count = 0;

  return function inner() {
    count++;
    console.log(count);
  };
}

const counter = outer();

counter(); // 1    // ye dono closure hain
counter(); // 2

👉 Here, inner() remembers count → this is closure
--------------------------------------------------------------



🔹 When is Closure Created?
👉 Closure is created:
✔ Whenever a function is created inside another function
✔ And it uses variables from outer function


⚠️ Important:
It is created at function creation time, not at execution


🧠 One-Line Definition : ----------------------------------------------------
👉 Closure = function + its remembered outer variables
------------------------------------------------------------------------------
🔥 Real-Life Use :
  * Data hiding (private variables)
  * Counters
  * Callbacks / event handlers

🔹Que 12. splice() vs slice() ?

🔸 splice() : -
👉 Original array ko change karta hai (modify)
👉 Add / Remove dono kar sakte hain

ex : --------------------------------------
const arr = [10, 20, 30, 40];
arr.splice(1, 2);
console.log(arr); // [10, 40]
_________________________________________________
🔸 slice() : -
👉 Original array ko change nahi karta ❌
👉 Sirf copy banata hai

ex : --------------------------------------
const arr = [10, 20, 30, 40];
const newArr = arr.slice(1, 3);
console.log(newArr); // [20, 30]
console.log(arr);    // [10, 20, 30, 40]
---------------------------------------------

🧠 One-Line Difference :
👉 splice = change original array
👉 slice = copy without changing

🔹Que 13 : map() vs forEach() ?

🔸 map() : -
👉 New array return karta hai ✔
👉 Data transform karne ke liye use hota hai

ex : --------------------------------------
const arr = [1, 2, 3];
const result = arr.map(num => num * 2);
console.log(result); // [2, 4, 6]
_________________________________________________

🔸 forEach() : -
👉 Kuch return nahi karta ❌
👉 Sirf loop / side work ke liye use hota hai

ex : --------------------------------------
const arr = [1, 2, 3];

arr.forEach(num => {
console.log(num * 2);
});

// Output: 2, 4, 6

🧠 One-Line Difference :
👉 map = transform + return new array
👉 forEach = loop + no return

🔹Que 14: Can an Object key be a number or boolean?

👉 Yes, but internally they become strings

🔸ex: ----------------------------------------------
const obj = {
    1: "one",
    true: "yes"
};
console.log(obj[1]);      // "one"
console.log(obj["1"]);    // "one"

console.log(obj[true]);   // "yes"
console.log(obj["true"]); // "yes"
🧠 Explanation (Simple)
-------------------------------------------------------------

👉 JavaScript object keys are always strings (or symbols)
👉 Number & boolean keys automatically convert into string

🔹Que 15: Reference Copy vs Deep Clone

🔸 Reference Copy ==============================================
👉 Same memory ko point karta hai (copy nahi, link hota hai)
ex : ----------------------------------------------
const obj1 = { name: "Nick" };
const obj2 = obj1;
obj2.name = "Rahul";

console.log(obj1.name); // Rahul ❗ (change ho gaya)
---------------------------------------------------
👉 Dono same object share karte hain


🔸 Deep Clone ===================================================
👉 Naya object banta hai (independent copy ✔)

ex: ----------------------------------------------
const obj1 = { name: "Nick" };
const obj2 = JSON.parse(JSON.stringify(obj1));
obj2.name = "Rahul";

console.log(obj1.name); // Nick ✔ (no change)
----------------------------------------------
👉 Original safe rehta hai


🧠 One-Line Difference :- 
👉 Reference = same object share
👉 Deep Clone = completely new copy
